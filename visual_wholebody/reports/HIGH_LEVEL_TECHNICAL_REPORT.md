# 高层模型技术报告

## 1. 范围与代码入口

本报告总结 `high-level/` 目录下高层策略系统的实现结构、训练机制与能力边界。当前代码基线围绕 B1+Z1 平台的多物体抓取任务展开，主路径可概括为：`train_multistate.py`（teacher PPO）与 `train_multi_bc_deter.py`（student DAgger）驱动 `envs/b1z1_pickmulti.py` 环境，并在 `envs/b1z1_base.py` 中完成高低层耦合控制。

高层输出并不直接下发到底盘和机械臂电机，而是先转换为末端目标与底盘速度命令，再交由低层策略和环境内 IK 控制链执行。系统因此属于分层控制：高层负责任务级决策与时序组织，低层负责动力学可执行性与全身稳定控制。

## 2. 任务定义与场景组成

当前公开代码仅包含“抓取多个物体（pick multi objects）”任务，不包含“放置到目标位姿（place）”任务。单个并行环境包含三个 actor：`robot`、`table`、`cube`。其中 `cube` 只是变量命名，实际可替换为配置中的杯子、碗、盒子等多类物体模型。

在任务语义上，每个环境同一时刻只有一个待抓取目标，因此不存在“多目标中再选择一个”的决策分支。目标物体由环境创建阶段按对象列表索引分配，reset 期间主要随机化位置与朝向，而不是在同一环境里并列多个候选物体。

## 3. 感知与目标指定机制

系统在训练路径上分为两类策略。teacher 是状态策略，直接使用仿真真值物体状态；student 是视觉策略，使用分割与深度构造的视觉输入。两者都不使用自然语言输入，也不使用人工框作为策略输入格式。

视觉路径中的目标 mask 在仿真内通过 segmentation id 直接提取。实现方式是读取 Isaac Gym 的 `IMAGE_SEGMENTATION`，再用固定目标 id 做等值筛选。该机制是仿真特权能力，不等价于真实世界可直接获得的监督。

## 4. 动作空间与控制语义

高层默认动作维度为 9（可在特定选项下扩展）。其核心语义为：末端位置增量、末端姿态增量、夹爪开合命令、底盘前向与偏航命令。动作经过裁剪和尺度变换后，更新 `curr_ee_goal_cart` 与 `curr_ee_goal_orn_rpy`，并生成底盘命令 `commands`。随后系统调用低层策略与 IK 执行一个高低频耦合仿真步。

这意味着高层策略学到的是任务级意图轨迹，而非“直接关节级追踪”。行为质量由高层命令、低层策略稳定性、IK 解算可达性和物理仿真共同决定。

## 5. 观测结构：Teacher 与 Student

Teacher 观测由三部分构成：对象特征 `feature_obs`、机器人与物体状态观测 `robot_obs`、历史动作或历史命令。`feature_obs` 默认是 1024 维，由每个对象目录下的 `features.npy` 离线加载后绑定到环境实例。teacher 网络会先将该特征压缩，再与其余状态融合。

Student 观测使用 `states_buf`，由视觉历史与筛选后的本体状态拼接。视觉部分按模式组织（`full`、`wrist_seg`、`front_only`、`seperate`），默认是三帧历史堆叠。状态部分会显式移除 teacher 的特权对象信息（对象 feature 与对象真值位姿），保留末端状态、关节状态、命令、末端目标和动作历史，用于蒸馏学习。

从信息结构看，teacher 直接“看见目标真值”，student 只能“看见视觉结果 + 本体状态”。teacher 与 student 的性能差异与泛化边界，主要由这组信息不对称决定。

## 6. 奖励设计与隐式分阶段机制

高层奖励由配置中的多个分项线性加权求和。核心任务项是 `approaching`、`lifting`、`pick_up`，并辅以底盘稳定相关项与动作平滑项。代码中没有单独的 `stage_id` 状态机，但通过条件门控实现了隐式分阶段。

具体而言，接近阶段主要由 `approaching` 驱动；进入抬起后，`approaching` 会被关闭，`lifting` 与 `pick_up` 成为主信号；训练后期再引入更严格的保持逻辑（`hold_steps`）和部分底盘行为约束项。该机制本质是“reward 条件切换”，而不是显式状态机切换。

## 7. Reset 条件与训练数据分布

环境 reset 由多类条件触发：回合超时、机身姿态越界、机身高度异常、IK 失败、物体掉落、抓起后掉落，以及视觉模式下的额外几何约束。成功抓取也会触发 reset 或保持后 reset（取决于训练阶段）。

值得注意的是，视觉模式下没有“任意一路相机看不到目标就立即 reset”的显式规则。腕部相机短时丢目标不会单独触发 reset，系统通常继续推进，直到命中其他终止条件。这使训练中自然出现“部分可见/短时不可见”的样本。

## 8. 训练方式：PPO Teacher 与 DAgger Student

Teacher 使用 PPO 在线训练，环境 reward 通过 GAE 构造 returns 与 advantages，优化目标为标准 PPO 形式（clipped policy loss + value loss + entropy）。reward 对 teacher 是直接监督信号。

Student 使用 DAgger 在线蒸馏，训练时边与仿真交互边收集样本，并按 rollout 周期更新。student 的主损失是对 teacher 动作的 MSE 拟合，而非直接最大化环境 reward。reward 对 student 的作用是间接的：它先塑造 teacher，再通过 teacher 标签影响 student。

因此，系统整体不是离线先采集后训练，而是在线交互采样和在线参数更新交替进行。

## 9. 安全约束、碰撞与 IK 失败处理

高层对末端目标提供了盒形硬裁剪，可抑制明显越界目标点，但不等价于轨迹级安全约束。系统未提供强显式的“全程轨迹碰撞惩罚”或“机体自碰撞惩罚”主路径；部分接触惩罚代码存在但未启用。

IK 失败在当前配置下主要作为终止信号（early stop），并不默认对应单独的显式惩罚项。其学习效果主要来自“提前终止导致后续正回报缺失”的间接惩罚。这种信号能够抑制部分无效策略，但约束强度弱于显式安全损失，因此在边界工况下仍可能输出潜在风险目标点。

## 10. 单目/双目可见性与容错能力

代码支持固定单路模式（只前视或只腕视），也支持双路模式。若目标是“任意一路可见即可执行（OR 可见性）”，仅切换到固定单路模式并不等价。现有双路策略默认并未专门按“单路随机失效”进行鲁棒训练。

在工程上，若要实现 OR 可见性容错，需要在训练分布中加入视角缺失扰动（如随机遮蔽一路输入）并在推理时做可见性门控。否则直接使用双路训练模型，遇到一路长期不可见时性能通常会不稳定。

## 11. 部署侧差异与论文实现关系

仿真中 mask 来源是 segmentation 特权；真实部署必须由外部感知模块提供目标分割。论文部署路径采用 TrackingSAM（SAM + AOT）并在 reset 时做人工初始化标注，然后跟踪输出 mask。这一方案可行，但会引入遮挡、相似外观与视角变化下的跟踪脆弱性。

因此，仓库中的仿真训练接口与真实部署链路之间存在可见的感知域差距。该差距不是单一参数可消除的问题，而是整体感知-控制接口设计问题。

## 12. 结论

当前高层模型是一个面向抓取任务的分层控制系统，具备明确的工程可执行性：teacher 借助特权状态学习任务策略，student 通过视觉蒸馏获得可部署近似策略，系统能够完成“接近-抓取-抬起/持握”的主流程。

同时，其边界也较清晰：任务范围聚焦 pick-and-hold，尚未覆盖 place；安全约束以硬裁剪与终止为主，缺乏强轨迹级碰撞惩罚；视觉部署依赖外部分割与跟踪模块；对“任意一路可见即可执行”的鲁棒性需要专门训练策略支持。整体上，这是一套有效但任务特化、感知耦合较强的高层实现。
